***pdf3장 읽어보기***

 - column(칼럼) : 열
 - ' -- '  : sql구문에서의 한 줄 주석 기호.
 - sql구문은 대/소문자 구분 안 함. 단, 저장된 값(데이터)은 대/소문자 구분함.
 - 화살표가 가리키는 테이블 : 값을 제공하는 테이블
 - 새발바닥이 가리키는 테이블 : 값을 여러번 사용한다는 표시
 - DB에서 문자열은 반드시 작은 따옴표(' ')로 묶어서 표시함

 - 사용법
  - 1) 일반 사용자(개발자)가 데이터베이스를 사용하려면
   - 관리자로부터 사용자 계정과 암호를 받아야 함. 
   - 관리자가 사용자 계정과 암호를 만드는 구문
   >>> CREATE USER 사용자계정 IDENTIFIED BY 암호;(소문자로 써도 상관없음.)
   - 수업 진행을 위한 사용자 계정 만들기
   - 18C XE에서는 사용자 계정 글자 앞에 반드시 c## 을 붙여줘야 함.
   >>>create user c##student identified by c##student;  
   - 실행시킬 문장에 마우스 커서를 두고 ctrl + enter 누르면 실행됨.(해당 줄만 실행시킬 때)

 - 2) 관리자가 만든 사용자 계정에 대해 데이터베이스 접속 권한과 
      데이터베이스 자원을 사용할 수 있는 권한을 부여해야 함
   - 권한 부여 구문 :
	GRANT 권한종류/ROLE 이름 TO 사용자계정;
   >>> GRANT CONNECT TO c##student;
   >>> GRANT RESOURCE TO c##student;
   >>> create table test (t_id number); - cmd창에 테이블 만들기 테스트 실행

  - 3) 18C 에서는 권한 부여 후에 데이터 기록을 위한 테이블 스페이스 할당을 해야 함.
   >>> ALTER USER c##student QUOTA 1024M ON USERS;

 - 테이블 내에서 칼럼마다 데이터 타입과 값이 어떤 형태로 입력되어 있는지 잘 확인하는 것 매우 중요.
   >>> 열, 데이터 부분 꼼꼼히 확인

 - 제약조건(CONSTRAINT)
  - p : primary key, 기본키, 이로 설정한 값은 똑같이 두번 넣을 수 없음
  - u : unique, 같은 값을 두번 기록 못함
  - f : foreign key, 다른 테이블이 제공하는 키를 가져다 쓰는 키
  - c : check, 해당 칼럼에 조건을 부여함(ex) nullable)
